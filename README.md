# Vibe Coding Hackathon - Facilitator Kit

A 90-minute workshop where law firm trainees build real working apps using Claude Code - no programming experience needed. This repo is everything you need to run it.

## Make it yours

This kit ships with a worked-example identity so it runs out of the box: a fictional firm (**Nairo Partners LLP**), a fictional internal team (the **Innovation Team**), a self-contained design system, and synthetic firm data in `mockdata/`. None of it is real. Before you run a session, swap in your own:

- **Branding** - colours, fonts, and the design-system label in `docs/DESIGN.md` and `starter.html`
- **Hosts** - the `[Host name] · [Role]` placeholders in `docs/OPENING_DECK.html`
- **Shipping detail** - `docs/SHIPPING_EXAMPLE.md` describes one firm's stack, data tiers, and approval path; replace it with yours
- **API keys** - copy `.env.example` to `.env` and add your own (the `.env` is gitignored; never commit it)

## Folder structure

```
/
├── CLAUDE.md              Session instructions - loaded automatically by Claude Code
├── starter.html           Blank HTML template with the Nairo Partners design system pre-loaded
├── docs-viewer.html       Open locally to browse the docs in a browser
├── docs/                  Facilitator reference - read before the session, don't edit during
└── example/               A complete example session, built during a test run
    ├── index.html         The app that was built (Trainee Litigation Simulator)
    ├── PLAN.md            The plan created by /kickoff
    ├── presentation.html  The slide deck generated by /present
    └── materials/         The scenario documents used in the exercise
```

The root folder is the **session template** - copy it fresh for each group. The `example/` folder shows what a finished session looks like.

## How to run a session

**Before the day:**
1. Copy this folder - one copy per group
2. Install [Claude Code](https://claude.ai/code) on each machine
3. Open the folder in Claude Code - `CLAUDE.md` loads automatically
4. Read [docs/FACILITATOR_GUIDE.md](docs/FACILITATOR_GUIDE.md)
5. If running a scenario-based session, drop source documents into a `materials/` folder

**On the day**, run the session in five phases:

| Time | Phase |
|---|---|
| 0-15 min | Discovery - work through [docs/DISCOVERY.md](docs/DISCOVERY.md) |
| 15-20 min | Design brief - complete [docs/DESIGN_BRIEF.md](docs/DESIGN_BRIEF.md), then run `/kickoff` |
| 20-55 min | Build - iterate with Claude |
| 55-70 min | Polish - tighten the UI, no new features |
| 70-75 min | Run `/present`, open `presentation.html` in the browser |
| 75-90 min | Present - 5 minutes per group |

**If a group arrives with a clear idea**, skip Discovery and Design Brief. Describe the idea to Claude and run `/kickoff` straight away.

## Slash commands

| Command | What it does |
|---|---|
| `/kickoff` | Creates `PLAN.md` from the design brief - starts the build |
| `/handover` | Saves session state to `HANDOVER.md` before closing the chat |
| `/pickup` | Restores full context at the start of a new chat |
| `/present` | Generates `presentation.html` - a slide deck for the demo |

## Design system

All apps should use the Nairo Partners design system defined in [docs/DESIGN.md](docs/DESIGN.md). When prompting Claude, include:

> "Use the Nairo Partners design system from docs/DESIGN.md"

Or copy `starter.html` as your starting point - it has the full CSS already loaded.
